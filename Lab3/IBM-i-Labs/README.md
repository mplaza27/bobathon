## IBM i and RPG Deep Dive Lab Exercises

Welcome to our collection of IBM i and RPG labs! In these lab exercises, you'll put Bob to work investigating IBM i and RPG processes by examining a sample Retail Application. You'll first investigate the sample retail application and use Bob to explain what it is and what its key components are, and then you'll use Bob to make a few modifications and generate a custom full-stack web application in which you can interactively track its inventories. Please follow the attached sample READme files for each interactive lab to get started. They are included in this folder as follows: 

1. Lab3.0-RPG-Project-Introduction.md
2. Lab3.1-RPG-Documentation.md
3. Lab3.2-UI-Modernization.md
4. Lab3.3-DDS-to-SQL-RLA-Refactoring.md

### Section 0: Prerequisites
- Bob installed and logged in
- IBM-i-Labs files downloaded on your VM
- Uncompress the zip file bob-rpg-ibm-i-main.zip
- Set your Bob IDE directory to the resulting file folder bob-rpg-ibm-i-main
- Now you're ready to follow those four lab guides above to perform the following labs!

### Section 1: Lab3.0-RPG-Project-Introduction
In this first lab, you will follow the lab guide Lab3.0-RPG-Project-Introduction.md to explore, investigate, and explain what is occuring in your example SAMCO application. You will use Bob to explore your file directories, their contents, what the sample SAMCO application is, what it does, and what its key components are.

### Section 2: Lab3.1-RPG-Documentation
Now you will make a few changes to the SAMCO application in the lab guide Lab3.1-RPG-Documentation.md, with Bob's help. You'll investigate the sample application further using Bob and see how its code modernization capabiltiies in RPG are put into action.

### Section 3: Lab3.2-UI-Modernization
In this section of the lab, you'll follow Lab3.2-UI-Modernization.md to construct a frontend UI end-to-end with Bob's help in just minutes. You'll create an interactive React Carbon website which you'll then be able to use to trak the inventory in your SAMCO application in real-time.

### Section 4: Lab3.3-DDS-to-SQL-RLA-Refactoring
Finally, in this third hands-on lab exercise, you'll follow Lab3.3-DDS-to-SQL-RLA-Refactoring.md to use IBM Bob to convert a single Record Level Access (RLA) operation to modern SQL. In just minutes, you'll have refactored an entire operation to now use a more up-to-date SQL architecture.
